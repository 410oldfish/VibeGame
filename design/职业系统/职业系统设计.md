# 《VibeGame》职业系统设计

> 文档状态：草案 v0.1
> 关联总案：[`../游戏粗略策划案.md`](../游戏粗略策划案.md)

## 1. 文档定位

本文件是职业系统的主文档，负责定义职业在整个游戏中的共性结构，以及职业系统与构筑、战斗、地形、奖励等系统的连接方式。

本文件负责：

- 职业选择在游戏流程中的位置。
- 职业的共用组成要素。
- 职业与牌组、技能、属性、成长之间的关系。
- 单职业文档的统一记录方式。

本文件不负责：

- 某个职业的具体卡牌数值。
- 某个职业的完整技能条目细节。
- 战斗中的公共回合时序与空间规则。

## 2. 设计目的

- 让每个职业都具备清晰可识别的战术身份。
- 让职业差异真正体现在位置、地形、敌方意图和资源使用方式上。
- 让单职业文档可以围绕统一结构持续扩展，而不是各写各的。

## 3. 系统边界

职业系统位于“玩家进入一局游戏”与“玩家在战斗中如何行动”之间，决定玩家能带着什么身份、资源和成长方向进入其他系统。

| 类型 | 主要内容 |
| --- | --- |
| 上游依赖 | 角色选择入口、构筑规则、奖励投放、状态词条、地形与敌人机制 |
| 系统产出 | 初始属性、初始牌组、职业技能、关键词、成长方向、职业专属奖励池 |
| 下游影响 | 玩家战斗风格、构筑路线、地形使用方式、队伍协作定位 |

## 4. 系统规则

### 4.1 职业选择

- 游戏开始时，每个玩家先选择一个职业。
- 职业选择确定角色的基础属性、初始牌组、技能结构和后续可获得内容池。
- 多人模式下，各玩家可以选择不同职业，以形成不同的战场职责分工。

### 4.2 职业的共用组成

每个职业至少由以下部分组成：

| 组成 | 说明 |
| --- | --- |
| 初始属性 | 生命、能量、体力等基础战斗资源 |
| 关键词 | 描述该职业的主玩法语言与设计方向 |
| 技能 | 主动或被动的职业能力，强化其战术身份 |
| 初始牌组 | 决定开局可执行策略和构筑起点 |
| 成长方向 | 职业在奖励选择后可能分化出的路线 |

### 4.3 职业身份约束

- 职业系统应优先定义“怎么打”，而不是只定义“数值高低”。
- 职业强项应与战场位置、地形互动或敌方意图应对方式直接相关。
- 单职业的弱项应真实存在，从而为队友配合和构筑修正留出空间。

## 5. 核心机制

### 5.1 身份由系统交互决定

职业身份不只由技能标签决定，还由其在其他系统中的表现共同构成：

- 在战斗系统中，职业决定行动风格与队伍职责。
- 在卡牌构筑系统中，职业决定初始牌组与成长路线。
- 在地形系统中，职业决定更擅长建场、破场、守场还是借场输出。
- 在敌人系统中，职业决定更擅长规避、打断、诱导还是压制敌方牌局。

### 5.2 共性规则与个体条目分离

职业系统主文档只写所有职业共用的组织方式；具体职业条目单独维护。这样后续新增职业时，只需要遵守统一模板，而不用重复解释整套职业系统。

## 8. 约束与例外

- 本文不记录单职业完整数值与完整牌表。
- 若未来加入副职业、双职业或角色外观系统，应在职业系统之外单独建文档处理，不要回灌到职业共性结构里。
- 若后续职业成长被做成局外长期系统，需要新增成长接口文档，而不是在单职业文档中硬塞。
