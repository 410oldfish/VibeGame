# 《VibeGame》粗略策划案

> 文档状态：概念草案 v0.3
> 用途：作为唯一总纲文档，统一核心玩法方向，并索引按系统拆分后的设计文档。
> 原则：本文只保留设计支柱、核心循环、系统地图和高层边界；具体规则、子系统、内容原型和验证记录全部写入对应系统文档。

## 1. 项目概述

### 1.1 一句话描述

一款以六边形战场为核心的多人合作卡牌 Roguelike：玩家在持续变化的地形中移动、出牌、改造战场，并与同样依靠牌组行动的敌人进行可预判、可干涉的策略对决。

### 1.2 目标体验

- 将位置与移动提升为和能量同等重要的回合资源。
- 让敌人也拥有“打牌感”，使其行动具备预测空间与干涉空间。
- 让地形从场景背景升级为可被生成、破坏、转化和组合利用的战斗资源。
- 让职业构筑围绕位置、地形与状态形成清晰的战术身份。
- 以合作模式为当前设计基础，强化队友之间的站位、保护与连携。

### 1.3 类型与形式

| 项目 | 设定 |
| --- | --- |
| 类型 | 战术移动 + 卡牌构筑 + Roguelike 冒险 |
| 战斗视角 | 六边形网格战场，回合制 |
| 局外结构 | 三章节路线地图，每章包含事件、商店、休息与战斗节点 |
| 当前多人方向 | 玩家合作对抗敌方阵营，房主权威结算 |
| 失败条件 | 玩家阵营所有角色死亡 |
| 通关条件 | 击败第三章首领 |

## 2. 设计支柱

### 2.1 每张牌都应与位置有关

玩家需要在“打出高收益卡牌”和“移动到有利位置”之间权衡资源。攻击距离、攻击方向、击退落点、地形覆盖范围与队友站位都应改变卡牌价值。

### 2.2 敌人也在进行一局牌局

敌人不是固定轮播技能的木桩。敌人有牌组、有抽取结果、有弃牌循环，并通过可读意图向玩家施压；玩家则可以躲避、打断、诱导或污染敌方牌局。

### 2.3 战场可以被重写

格子的价值会在战斗中持续变化。燃烧、水洼、藤蔓、陷坑、掩体等地形既影响移动与伤害，也能被玩家和敌人转化、覆盖和组合利用。

### 2.4 构筑应服务于战术身份

牌组强化的不只是伤害数字，还包括玩家选择战场规则的方式，例如冲撞推墙、护卫阵线、植物领域、属性地表、位移连击或控制敌方手牌。

## 3. 核心循环概述

### 3.1 单局冒险循环

选择职业与初始牌组后，玩家沿章节路线推进，在战斗、事件、商店和休息处之间进行取舍，并通过卡牌、道具、装备与系统能力成长，最终挑战章节首领。

### 3.2 单场战斗循环

玩家先观察战场地形与敌方公开信息，再在玩家回合内完成抽牌、恢复资源、移动、出牌、使用技能/道具和改装地形；随后敌方按牌组与意图执行行动，并结算持续状态与场地变化。

### 3.3 每回合的关键决策

玩家每回合需要同时处理三类问题：

1. 牌局决策：当前是打出收益、保留关键牌，还是为下回合铺垫。
2. 空间决策：体力要用于接近、脱离、争夺关键格，还是触发位移组合。
3. 环境决策：是立即利用现有地形，还是投入资源改装战场以换取后续优势。

## 5. 当前高层边界

在当前系统划分下，几个关键边界定义如下：

- 战斗系统只定义战斗公共时序、共享行动窗口和胜负结算，不再承担牌组、奖励、地图和职业的细节规则。
- 六边形战斗规则作为战斗系统的空间子系统，只定义位置、路径、占位和范围表达。
- 卡牌构筑系统负责牌组生命周期和资源关系；职业系统负责职业共性和单职业条目。
- 状态效果系统负责跨系统语义，状态表只做词条数据，不再承担系统解释。
- 遭遇系统和原型验证文档不再作为底层规则源，只负责内容落地和验证记录。
- 联机系统只负责同步边界与实现承载，不反向定义玩法规则。

## 6. 当前新增重点

现阶段总案重点关注以下方向：

- 六边形战斗空间的设计边界与价值表达。
- 敌人卡组、公开意图和玩家干涉敌方牌局的机制框架。
- 地形改装与地形连锁在战斗中的可读性与收益感。
- 可玩原型的范围控制、测试指标与后续验证问题。

## 7. 待确认的高层问题

- 游戏首发是否以多人合作为主，单人模式如何承接同一套系统。
- 战场是否引入朝向、侧击/背击和完整视线遮挡，或先维持轻量空间规则。
- 敌方牌组信息公开到什么程度，以及玩家干涉敌方牌局的能力投放给哪些职业或系统。
- 强地形效果、地形改装来源和战斗场地生成方式如何在原型后定版。

## 8. 文档维护原则

- 总纲只记录方向、边界和系统之间的关系。
- 每个系统主文档使用统一结构：文档定位、设计目的、系统边界、系统规则、核心机制、子系统、依赖关系、约束与例外、相关文档。
- 子系统细节、资源表、敌人与职业条目、原型内容和验证记录全部维护在对应系统目录下。
- 当新增一个机制时，优先判断它属于哪个系统，再决定写入主文档、子系统文档还是验证文档。
