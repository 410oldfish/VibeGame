# 《VibeGame》原型数值与测试观察点

> 文档状态：草案 v0.2
> 文档定位：本文件属于“平衡与验证”层，不是玩法系统规则源。
> 关联总案：[`../游戏粗略策划案.md`](../游戏粗略策划案.md)

## 1. 文档目的

- 为首个可玩原型提供可快速落地的初期数值假设。
- 明确测试时需要观察的关键指标与主观反馈点。
- 汇总仍需在原型或策划会议中拍板的关键问题。

## 2. 当前验证范围

本文件主要用于验证以下系统是否成立：

- 战斗系统。
- 敌人系统。
- 地形系统。
- 卡牌构筑系统与职业系统的初始适配。

## 3. 初期数值假设

下列为便于搭建原型的假设，不视为最终规则：

| 项目 | 初期假设 | 需观察的问题 |
| --- | --- | --- |
| 玩家能量 | 沿用职业文档基础值 `3` | 地形牌是否挤压基础攻防出牌空间 |
| 玩家体力 | 沿用职业文档基础值 `3` 至 `4` | 普通移动是否太贵，导致玩家不愿换位 |
| 场地尺寸 | 约可容纳 `25` 至 `45` 个有效六边形格 | 是否既有绕行空间又能频繁产生接触 |
| 敌方意图展示 | 提前展示下一次执行的主要牌与覆盖格 | 信息是否足够规划而不完全消除风险 |
| 危险地形数量 | 初始危险格不超过有效格的约四分之一 | 地形是否影响决策但不限制可行动区域 |
| 地形牌占比 | 初始牌组中少量提供，奖励中逐步出现 | 玩家是否会主动将其视为构筑核心 |

## 4. 测试记录重点

测试时应记录：

- 每回合玩家平均移动格数与体力剩余量。
- 玩家是否能在敌人行动前理解危险范围和应对手段。
- 地形改装牌的选择率、打出率和造成的有效收益。
- 因地形/位移造成的击杀占比，以及其是否显著提高满足感。
- 合作模式中玩家是否容易互相堵位、误伤连锁或产生等待时间。

## 5. 待确认事项

以下问题需要在原型或后续策划会议中拍板：

1. 游戏首发是否以多人合作为主要模式，单人模式是否同时提供队友控制或平衡调整。
2. 战场是否引入单位朝向、侧击/背击和完整视线遮挡，或仅保留距离与直线规则。
3. 敌方牌组信息公开到什么程度：仅意图牌、弃牌堆，还是允许查看完整牌库摘要。
4. 玩家干涉敌方牌组的能力是通用系统，还是限定给某一职业、副职业或少量稀有牌。
5. 被推入深坑等强地形效果对首领与精英如何折算，避免地形流被免疫或过强。
6. 地形改装是主要由职业卡牌提供，还是额外引入所有角色通用的“工具/工程”系统。
7. 每场战斗场地为预制关卡、规则化随机生成，还是两者结合。
8. 三个现有职业的卡牌配置如何适配地形标签、敌方意图和六边形范围表达。

## 6. 使用原则

- 每轮原型测试只集中验证少量变量，避免多个系统同时改动导致结论失真。
- 数值记录应同时包含可量化指标与玩家对“清晰度”“满足感”“压力感”的主观反馈。
- 当测试目标是验证系统可读性时，应优先控制复杂度，而不是追求内容丰富度。
