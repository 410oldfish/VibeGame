# 《VibeGame》六边形战斗规则

> 文档状态：草案 v0.2
> 所属系统：[`战斗系统.md`](./战斗系统.md)

## 1. 文档定位

本文件是战斗系统的空间子系统文档，负责定义六边形战场中的距离、占位、路径、位移与范围表达。

## 2. 设计目的

- 让位置、路径与包围关系成为每回合都重要的决策变量。
- 用六边形网格降低距离与方向判定的歧义。
- 让击退、拉拽、守点、换位等玩法有统一的空间规则基础。

## 3. 规则范围

本文件负责：

- 六边形距离与方向语言。
- 单位占位与路径合法性。
- 主动移动与被动移动的空间差异。
- 位移引发的冲突与碰撞处理。
- 卡牌范围标签的表达方式。

本文件不负责：

- 敌方意图公开与执行。
- 地形词典与地形反应表。
- 牌组、资源和奖励投放规则。

## 4. 空间规则

### 4.1 采用六边形网格的目的

- 每格拥有六个相邻方向，减少正方网格中对角距离与阻挡的歧义。
- 增加围攻、扇形攻击、环形控制和击退路线的可能性。
- 更适合表现玩家围绕大型敌人移动、布置地形与形成防线的战术。

### 4.2 基础空间规则

| 规则项 | 当前方向 |
| --- | --- |
| 单格占位 | 通常一格最多存在一个单位 |
| 主动移动 | 每移动一格消耗 `1` 体力，且必须存在可通行路径 |
| 被动移动 | 击退、拉拽、冲撞等不消耗体力，但仍触发移动和地形效果 |
| 单位冲突 | 可移动单位可能被挤出；不可移动单位提供阻挡 |
| 范围表达 | 使用“六边形距离 N 格”“邻格”“直线方向”“扇形/环形”等统一术语 |
| 视线与遮挡 | 原型阶段只建议对远程指定类卡牌启用 |
| 朝向 | 作为待验证扩展规则，不作为当前基础前提 |

### 4.3 卡牌的空间标签

| 标签 | 含义 | 示例用途 |
| --- | --- | --- |
| 近战 | 仅能指定邻格单位 | 高伤攻击、格挡反击 |
| 射程 N | 可指定距离 N 以内单位 | 弓箭、法术、治疗 |
| 直线 | 沿六个方向之一生效 | 冲锋、剑气、射线 |
| 扇形 | 覆盖朝向附近多个格子 | 横扫、喷火 |
| 环形 N | 以施术者或目标为中心作用 | 嘲讽、爆炸、恢复 |
| 位移 | 推、拉、换位、跳跃、冲撞 | 将敌人送入陷坑或危险地形 |
| 地块 | 指定空格或地形而非单位 | 生成墙、点燃水面、种植藤蔓 |

## 5. 核心机制

### 5.1 主动移动与被动移动

- 主动移动是玩家主动消耗体力对位置进行投资。
- 被动移动不消耗体力，但依然改变站位关系，因此仍应触发对应的地形和移动机制。

### 5.2 位移与冲突处理

当位移导致同格冲突时，应优先根据“单位是否可移动”处理：

- 两个可移动单位：后进入格子的单位占据该格，原单位被挤出至最近可用空格。
- 一方可移动、一方不可移动：可移动单位在进入前终止移动。
- 双方都不可移动：视为非法状态，需要在实现层防止。

### 5.3 共享玩家回合下的空间配合

多人合作中的空间决策重点包括：

- 谁先改造地形、谁随后利用地形。
- 谁占据关键格保护低血量或蓄力中的队友。
- 是否用击退、拉拽或换位修正队友的危险位置。

## 6. 与战斗系统的接口

- 战斗系统调用本文件定义的移动、路径和占位规则。
- 地形系统会进一步限制可通行路径并改变落点价值。
- 敌人系统通过公开意图把空间问题暴露给玩家。
- 状态效果系统会改变单位是否可移动、是否受位移影响。

## 7. 约束与待验证项

- 原型优先验证距离、路径、位移和范围表达的清晰度。
- 原型阶段应谨慎引入完整视线遮挡与朝向规则。
- 若后续加入大型单位、多格占位或更复杂碰撞，需要新增空间扩展文档。
