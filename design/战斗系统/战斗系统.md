# 《VibeGame》战斗系统

> 文档状态：草案 v0.2
> 关联总案：[`../游戏粗略策划案.md`](../游戏粗略策划案.md)

## 1. 文档定位

本文件是战斗系统的主文档，负责定义一场战斗如何开始、如何推进、如何结算，以及战斗系统与其他系统的交界边界。

本文件负责：

- 战斗进入与结束条件。
- 玩家阵营与敌方阵营的公共回合结构。
- 共享行动窗口、结算顺序和胜负判定。
- 多人合作下的共同玩家回合规则。

本文件不负责：

- 六边形距离、路径、占位和碰撞的具体空间规则。
- 地形词典、地形反应和改装动作。
- 敌人牌组与意图的详细生成逻辑。
- 奖励数值、商店内容和章节路线结构。

## 2. 设计目的

- 让一场战斗拥有稳定、可读、可同步的公共时序。
- 让多人合作中的操作顺序和战斗结算具备一致规则。
- 让位置、卡牌、敌方意图和地形改装能在同一回合框架内协同工作。

## 3. 系统边界

战斗系统并不直接创造所有战斗内容，而是接收其他系统提供的战斗输入，并产出战斗结果。

| 类型 | 主要内容 |
| --- | --- |
| 上游依赖 | 玩家角色、玩家牌组、当前生命与资源上限、战斗场景、地形布局、敌人单位、敌方牌组、遭遇规则 |
| 战斗过程 | 回合开始、资源恢复、玩家行动、敌方执行、持续状态结算、胜负判断 |
| 下游结果 | 战斗胜负、生命变化、临时卡牌清理、奖励发放入口、章节推进或冒险失败 |

## 4. 系统规则

### 4.1 战斗进入

- 玩家从冒险系统进入战斗节点时，根据当前遭遇配置加载战场、敌人、环境规则与初始布局。
- 玩家角色带着局外生命值、当前职业构成和当前牌组进入战斗。
- 战斗中的临时卡牌、临时效果和临时地形改装默认不跨战斗保留，除非其他系统明确例外。

### 4.2 战斗开始阶段

战斗开始时建议按以下顺序初始化：

1. 生成战场、单位与环境规则。
2. 初始化玩家与敌人的牌组、资源和状态。
3. 公开战斗开始时应可见的敌方信息。
4. 进入首个玩家回合。

### 4.3 阵营回合结构

战斗采用“玩家阵营回合 -> 敌方阵营回合”的交替结构，当前默认玩家阵营先手。

| 阶段 | 主要结算 |
| --- | --- |
| 回合开始 | 结算回合开始时机的状态与环境效果，并检查胜负 |
| 资源恢复 | 恢复能量、体力、抽牌等本回合可用资源 |
| 玩家行动窗口 | 玩家阵营在共享回合中移动、出牌、用技能、用道具、改装地形 |
| 回合结束 | 处理保留、弃牌、持续效果和回合结束结算 |
| 敌方执行窗口 | 敌方根据当前意图和可执行条件行动 |
| 再判断 | 结算战场变化并重新检查胜负与下一轮准备 |

### 4.4 共享玩家回合

- 多名玩家处于同一个玩家回合窗口内行动，不拆分为个人独立回合。
- 玩家可自由协商行动顺序，但实际结算顺序以服务器收到的请求顺序为准。
- 若表现层动画导致状态短时间不同步，仍以战斗结算层的最终顺序为准。

### 4.5 玩家出牌与体力移动窗口

- 玩家在共享玩家回合中可打出满足费用、目标、距离和其他释放条件的手牌。
- 卡牌开始结算后，必须完整完成该卡牌引发的伤害、状态、位移、地形和触发效果，再接受新的玩家操作。
- 在不存在尚未结算完成的卡牌效果时，玩家可以发起一次消耗体力的主动移动。
- 每名玩家在自己的本回合内，至多发起一次消耗体力的主动移动；本回合一旦已经以该方式移动过，不能再次消耗体力发起主动移动。
- 该次主动移动仍按照路径中的移动格数消耗体力，能走多远由剩余体力、路径和空间规则共同决定。
- 由卡牌、技能、状态或外部效果产生且不消耗体力的位移，不视为已经使用本回合的体力移动机会，除非效果文本明确写明例外。

### 4.6 胜负结算

- 当敌方阵营被击败时，战斗胜利。
- 当玩家阵营所有角色死亡时，战斗失败。
- 战斗胜利后将结果移交给奖励与成长系统。
- 战斗失败后将结果移交给冒险系统，决定本局是否终止。

## 5. 核心机制

### 5.1 共享信息驱动决策

玩家在自己的行动窗口中，需要同时响应敌方已公开信息、当前地形状态和队友位置。战斗系统的核心不是单个动作，而是让这些信息在同一时序中成立。

### 5.2 多系统共同结算

战斗系统负责承载多个子系统的调用：

- 六边形空间决定单位能否到达目标位置。
- 卡牌构筑系统决定玩家有哪些手段可用。
- 敌人系统决定本轮敌方会如何行动。
- 地形系统决定战场会如何约束和奖励位置变化。
- 状态效果系统决定不同效果在什么时机生效。

### 5.3 战斗结果作为统一出口

战斗系统不直接定义奖励内容本身，而是统一产生战斗结果，再把结果交给奖励与成长系统、冒险系统与联机系统分别处理。

## 8. 约束与例外

- 本文只定义战斗公共框架，不再承担牌区细则、职业条目、商店服务和事件收益等跨系统细节。
- 六边形空间仍为当前战斗原型的关键前提，因此任何战斗规则若依赖距离或路径，均应回到空间子系统文档说明。
- 若未来加入异步结算、单人 AI 队友或更复杂的优先级链，需要补充新的战斗子系统文档。
