# 《VibeGame》状态、技能与道具联动规则

> 文档状态：草案 v0.2
> 关联总案：[`../游戏粗略策划案.md`](../游戏粗略策划案.md)

## 1. 文档定位

本文件是状态效果系统的主文档，负责定义状态、技能、道具和跨系统效果在战斗中的语义边界与结算关系。

本文件负责：

- 状态在系统层面的分类与用途。
- 关键时机点与结算顺序。
- 状态对敌人意图、地形、技能和道具的影响。
- 词条表与系统语义之间的对应关系。

本文件不负责：

- 每个状态的完整数据表。
- 技能与道具的职业具体数值。
- 地形词典、敌方牌组词典和战斗时序本体。

## 2. 设计目的

- 让现有状态词条直接服务于位置、地形和敌方意图三大核心机制。
- 让技能、状态与道具在不同系统之间使用统一的解释框架。
- 减少“已展示意图”和“尚未执行效果”之间的规则争议。

## 3. 系统边界

| 类型 | 主要内容 |
| --- | --- |
| 上游依赖 | 战斗时序、敌方意图、地形规则、职业技能、道具来源 |
| 系统产出 | 状态生效结果、目标可行动性变化、地形/意图修正、技能与道具的统一语义 |
| 下游影响 | 玩家回合决策、敌人执行结果、地形反应时机、联机同步事件类型 |

## 4. 系统规则

### 4.1 核心联动状态

现有增益状态与减益状态词条已覆盖大量战术效果，其中与新核心机制最直接相关的状态包括：

| 系统 | 相关状态 | 用途 |
| --- | --- | --- |
| 移动控制 | 稳固、飞行、束缚、缠绕、残废、恐惧、诱惑 | 改变路径、位移收益和地形风险 |
| 防御站位 | 格挡、圣盾、戒备、嘲讽、隐身 | 保护关键格或操纵敌方目标 |
| 地形属性 | 燃烧、感电、中毒、石化、无实体 | 支撑地表反应与伤害类型 |
| 敌方出牌干涉 | 眩晕、冰冻、沉默、封印、混乱 | 打断或改变敌方公开意图 |

### 4.2 技能与道具的系统语义

- 技能应优先放大职业在移动、地形改装或敌方意图干涉方面的战术身份。
- 道具应作为跨职业补足手段，让非专精职业也能有限度地处理环境问题。
- 同一个关键词在卡牌、技能和道具中的结算语义应尽量保持一致。

### 4.3 待统一的争议规则

后续需要优先统一以下交界问题：

- 打断类状态是取消本次意图、延后执行，还是改为弃牌。
- 沉默、封印、混乱等效果影响已公开意图，还是只影响下次生成。
- 强制位移导致目标或施术者脱离合法范围时，敌方牌如何结算。
- 地形伤害、持续状态与意图执行的先后顺序如何统一。

## 5. 核心机制

### 5.1 状态词条表与系统语义分离

状态表负责记录词条本身；本文件负责解释词条在战斗系统、敌人系统和地形系统中的统一含义。这样才能避免词条表不断膨胀成大而全规则桶。

### 5.2 效果时机是跨系统接口

例如“回合开始时”“进入地形时”“执行意图前”这类时机点，实际上是多个系统之间的接口协议。只要这些时机点没有统一，系统之间就会反复争议。
