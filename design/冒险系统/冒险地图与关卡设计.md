# 《VibeGame》冒险地图与关卡设计

> 文档状态：草案 v0.2
> 关联总案：[`../游戏粗略策划案.md`](../游戏粗略策划案.md)

## 1. 文档定位

本文件是冒险系统的主文档，负责定义章节推进、路线选择、节点结构和战斗场景生成入口。

本文件负责：

- 章节结构与路线地图。
- 节点类型与节点职责。
- 章节环境主题的组织方式。
- 战斗场景在进入战斗前需要准备的关卡信息。

本文件不负责：

- 战斗中的公共时序与空间规则。
- 奖励数值、商店价格或原型数值假设。
- 敌人牌组的具体行为逻辑。

## 2. 设计目的

- 让局外路线选择与局内战斗环境形成连续的策略节奏。
- 让每章都具备逐步强化的环境主题。
- 让玩家在节点选择阶段就开始思考后续战斗、奖励与风险。

## 3. 系统边界

冒险系统位于“开始一局游戏”与“进入具体战斗或服务节点”之间，负责组织整局流程的外层结构。

| 类型 | 主要内容 |
| --- | --- |
| 上游依赖 | 职业选择结果、章节配置、节点池、关卡主题、奖励系统支持的服务类型 |
| 系统产出 | 下一节点选择、战斗场景参数、节点事件入口、章节推进结果 |
| 下游影响 | 战斗系统收到什么战场、奖励系统在什么节点发放什么内容、遭遇系统如何编排内容 |

## 4. 系统规则

### 4.1 章节路线结构

- 每章沿用 `10 行 4 列` 的纵向路线结构。
- 玩家团队共同选择路线，不拆分为多个小队。
- 每章最后一个节点汇入首领战斗节点。

### 4.2 节点类型

| 节点类型 | 主要作用 |
| --- | --- |
| 普通战斗 | 提供基础战斗学习、资源获取和卡牌奖励 |
| 精英战斗 | 检验构筑方向，提供更高风险和更高收益 |
| 首领战斗 | 作为章节终局，集中检验阶段性成长结果 |
| 商店 | 提供卡牌、装备、道具与构筑修正服务 |
| 特殊事件 | 提供高波动的长期收益、代价或规则变化 |
| 休息处 | 提供恢复、强化和战前准备空间 |

### 4.3 战斗场景生成入口

进入战斗节点时，冒险系统需要为战斗系统准备以下内容：

- 地图轮廓。
- 固定地形。
- 互动设施。
- 单位出生区域。
- 环境规则。

## 5. 核心机制

### 5.1 路线选择驱动构筑变化

冒险系统的价值不只是“把节点排成一张图”，而是让玩家因为未来要打什么、会得到什么、是否需要修正构筑而产生路线抉择。

### 5.2 章节主题驱动学习曲线

- 章节前段介绍本章环境规则。
- 章节中段通过精英、事件和商店强化玩家对该主题的应用。
- 章节末段通过首领战进行章节总结。

## 8. 约束与例外

- 本文不定义战斗内的移动、敌方意图、状态结算和奖励数值细节。
- 若未来加入分队、岔线剧情或章节外地图结构，需要单独扩展冒险系统而不是写入总纲。
- 场景生成只定义生成输入，不负责具体战斗内结算。
