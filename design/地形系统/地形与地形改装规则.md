# 《VibeGame》地形与地形改装规则

> 文档状态：草案 v0.2
> 关联总案：[`../游戏粗略策划案.md`](../游戏粗略策划案.md)

## 1. 文档定位

本文件是地形系统的主文档，负责定义战场地形的组织方式、地形改装动作和地形对其他系统的影响边界。

本文件负责：

- 地形分层模型。
- 基础地形池与临时覆盖层。
- 地形改装动作。
- 地形反应与封路约束的设计边界。

本文件不负责：

- 六边形路径本身的公共定义。
- 某个职业如何具体利用地形。
- 奖励投放节奏和章节场景编排。

## 2. 设计目的

- 让地形从静态障碍升级为可生成、破坏、转化和组合利用的战斗资源。
- 让玩家能够通过提前铺设和临场改造创造优势回合。
- 让地形与移动、敌方意图、状态和构筑路线形成稳定联动。

## 3. 系统边界

| 类型 | 主要内容 |
| --- | --- |
| 上游依赖 | 战场空间、关卡主题、职业与卡牌能力、敌方改场手段、状态结算规则 |
| 系统产出 | 通行限制、落点价值、环境伤害、区域收益、改装机会 |
| 下游影响 | 玩家移动路线、敌人意图质量、状态触发、构筑收益类型 |

## 4. 系统规则

### 4.1 地形的三层结构

为避免组合规则失控，建议一个格子最多由三层信息共同决定：

| 层级 | 含义 | 示例 |
| --- | --- | --- |
| 基础地面 | 决定基本通行和固有属性 | 普通地面、水域、高台、深坑 |
| 构筑物 | 可生成/破坏的实体，占位或提供遮挡 | 石墙、木箱、荆棘、炮台 |
| 临时覆盖 | 具有持续回合或传播规则的效果 | 火焰、毒雾、冰面、圣域、电场 |

同一层通常只保留一种状态；跨层反应通过明确规则表处理。

### 4.2 基础地形池

| 地形 | 移动影响 | 战斗影响 | 改装或组合方向 |
| --- | --- | --- | --- |
| 普通地面 | 正常通行 | 无 | 可建造、种植或覆盖 |
| 墙体/残骸 | 不可通行 | 阻挡部分视线与位移 | 可破坏；撞墙造成额外伤害 |
| 高台 | 需路径进入或消耗额外移动 | 远程攻击可获得收益，具体数值待测 | 可坍塌为残骸 |
| 深坑 | 不可主动进入 | 被推入时造成高额伤害或坠落效果 | 可临时搭桥或填平 |
| 水域 | 额外限制移动或冲撞 | 与电、冰、火产生交互 | 冰冻、通电、蒸发 |
| 泥地/毒沼 | 移动受阻 | 施加残废或中毒 | 可净化、点燃或种植 |
| 植物地 | 通行 | 德鲁伊和藤蔓效果强化 | 可生长为阻挡物或被焚毁 |

### 4.3 临时覆盖效果

| 覆盖 | 建议效果 | 典型反应 |
| --- | --- | --- |
| 火焰 | 进入或回合开始时获得燃烧 | 水域使火焰熄灭或生成蒸汽遮挡 |
| 冰面 | 移动结束后可能沿方向滑行一格 | 火焰融化为水域 |
| 电场 | 进入或停留时受到感电影响 | 水域扩大影响区域 |
| 毒雾 | 回合结束时施加中毒 | 火焰可引爆并清除毒雾 |
| 圣域 | 友方获得恢复/净化或敌方受到削弱 | 被污染效果覆盖或破坏 |

### 4.4 地形改装动作

| 操作 | 含义 | 示例 |
| --- | --- | --- |
| 生成 | 在满足条件的格子创造新内容 | 种下一块植物地；升起一面石墙 |
| 覆盖 | 为地面附加临时区域效果 | 点燃三格直线；生成圣域 |
| 转化 | 将现有地形换为另一种地形 | 水域冻结；毒沼净化 |
| 破坏 | 清除构筑物或危险覆盖 | 击碎墙体；扑灭火焰 |
| 推动 | 改变可互动构筑物的位置 | 将箱子推入通道形成掩体 |
| 引爆 | 消耗地形换取即时强效果 | 引燃毒雾；摧毁藤蔓造成束缚 |

## 5. 核心机制

### 5.1 地形是可运营资源

地形的价值不在于“场景看起来复杂”，而在于它能改变路径、改变风险、改变技能价值，并让玩家提前投资换来后续优势。

### 5.2 改装动作必须有限且可读

地形系统的组合空间很大，因此需要把可执行的改装动作收束为有限、统一、可预览的操作集合，避免系统失控。

### 5.3 地形收益要能支持构筑路线

地形体系不应只是一套场景机关。它应支持至少以下构筑乐趣：

- 击退陷阱流。
- 火焰传播流。
- 防线堡垒流。
- 生长领域流。
- 水电控制流。

## 8. 约束与例外

- 战场上必须始终保留可到达的有效战斗区域，避免双方被永久隔离。
- 即时致死的坑落、连锁爆炸等效果需要有明确预览与敌我一致的规则。
- 大范围持续地形应有持续时间、清除手段或覆盖上限。
- 首个原型不追求完整元素模拟，优先验证可读性和收益感。
