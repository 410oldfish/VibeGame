# 《VibeGame》卡牌与构筑系统设计

> 文档状态：草案 v0.2
> 关联总案：[`../游戏粗略策划案.md`](../游戏粗略策划案.md)

## 1. 文档定位

本文件是卡牌与构筑系统的主文档，负责定义玩家牌组的生命周期、牌区流转、资源关系以及构筑成长方向。

本文件负责：

- 玩家牌组在战斗中的进入、流转与清理规则。
- 牌区的职责划分。
- 能量、体力、生命与防御资源之间的关系。
- 构筑标签与构筑成长方向。

本文件不负责：

- 职业共性规则与职业选择入口。
- 战斗公共时序与空间规则。
- 奖励池的具体投放节奏。

## 2. 设计目的

- 让构筑成长服务于战术身份，而不是单纯堆叠伤害数字。
- 让玩家每回合同时考虑卡牌价值与空间价值。
- 让牌组成长能够围绕位置、地形和敌方牌局形成不同路线。

## 3. 系统边界

卡牌与构筑系统位于“玩家拥有什么能力”和“玩家在战斗里如何使用这些能力”之间。

| 类型 | 主要内容 |
| --- | --- |
| 上游依赖 | 职业选择、奖励投放、状态与技能效果、战斗公共时序 |
| 系统产出 | 玩家牌组、牌区状态、资源消耗关系、构筑成长方向 |
| 下游影响 | 战斗系统中的出牌组合、地形系统中的改装手段、敌人系统中的干涉能力 |

## 4. 系统规则

### 4.1 玩家牌组生命周期

- 玩家进入战斗时，以当前局外牌组初始化战斗牌组。
- 战斗中获得的临时卡牌默认只在本场战斗生效。
- 战斗结束后，临时牌与战斗期临时变化默认清除，局外牌组回到冒险层继续使用。

### 4.2 牌区划分

| 牌区 | 作用 |
| --- | --- |
| 抽牌堆 | 存放当前待抽取的卡牌 |
| 手牌 | 当前可使用的卡牌 |
| 弃牌堆 | 已打出或回合结束被弃掉的卡牌 |
| 消耗牌堆 | 因消耗、虚无等机制暂离本场循环的卡牌 |

基础流转方向：

- 抽牌堆 -> 手牌
- 手牌 -> 弃牌堆 / 消耗牌堆
- 弃牌堆 -> 抽牌堆（重洗）

### 4.3 卡牌功能维度

| 功能 | 作用 |
| --- | --- |
| 攻击牌 | 直接伤害、属性伤害、状态施加、击退与连击 |
| 防御牌 | 格挡、减伤、护卫、净化、移动救援 |
| 技能牌 | 抽牌、能量、体力、职业形态、敌方牌组干涉 |
| 移动牌 | 免费移动、跳跃、冲撞、交换位置、强制位移 |
| 地形牌 | 生成、转化、强化、破坏或引爆地形 |

“移动牌”和“地形牌”更适合作为设计标签，而不是绝对隔离的卡种。

### 4.4 资源关系

| 资源 | 主要用途 | 设计意义 |
| --- | --- | --- |
| 能量 | 出牌、主动技能 | 限制单回合卡牌组合上限 |
| 体力 | 主动移动、部分位移强化效果 | 限制位置调整次数，突出路线选择 |
| 生命 | 跨战斗生存资源 | 支撑路线抉择、恢复与风险收益判断 |
| 护甲/格挡 | 当前战斗内防御资源 | 允许玩家承受公开威胁或守住关键格 |

## 5. 核心机制

### 5.1 牌序与空间共同决定回合质量

玩家并不是只看“我抽到了什么牌”，还要看“我当前站在哪里、能不能移动到收益更高的位置、是否值得为地形投入资源”。这也是本系统和战斗系统强绑定的原因。

### 5.2 构筑路线服务职业身份

构筑强化的不只是伤害数字，还包括玩家选择战场规则的方式。典型方向包括：

- 冲撞与击退收益。
- 守点与反击。
- 地形生成与区域扩张。
- 敌方牌局干涉。

## 8. 约束与例外

- 本文只定义玩家牌组和构筑公共规则，不再承接职业共性、敌方牌组或奖励投放细节。
- 若未来加入更复杂的牌堆可视化、检索优先级或局外长期卡组管理，应继续拆出专门子文档。
- 资源数值本身属于平衡问题，不在本文件中定版。
